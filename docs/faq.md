# FAQ

## 为什么直接 `serve` 跑不起来

因为控制台认证配置还没初始化  
最常见的情况就是：

- `password_hash` 还是空的
- `session_secret` 还是空的

先跑：

```bash
./bares3d init
```

## 为什么分享链接还能 403

排查思路：

- 链接是不是过期了
- 链接是不是被 revoke 了
- 对应对象前缀是不是被桶策略明确 `deny` 了
- 对应桶或对象是不是已经没了

## 为什么我改了监听地址，但生成出来的链接还是不对

因为生成链接主要看的是：

- `public_base_url`
- `s3_base_url`

不是看 `listen.file` 和 `listen.s3`。  
监听地址是“进程绑哪”，基础地址是“外部该怎么访问你”

## 同步后为什么另一台机器没有我的 S3 凭据

因为 S3 凭据当前不在同步范围里  
这是预期行为

## 为什么对象在磁盘上的路径看起来和原始 key 不完全一样

因为对象 key 会先编码成安全路径段  
否则你迟早会遇到：

- `..`
- 保留文件名
- 特殊字符
- Windows 路径兼容问题

这是计划的一部分

## BareS3 更像对象存储还是文件服务

两者都是，不过我更倾向于像 MinIO 那样的产品定位  
只不过存储方式不同

所以你可以把它理解成：

- 上层是对象存储语义
- 底层尽量保持文件系统友好
